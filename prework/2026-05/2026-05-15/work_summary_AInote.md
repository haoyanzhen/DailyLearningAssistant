## 主要工作主题

### AI Agent 的上下文管理

本次提交把“上下文”从普通提示词材料提升为 Agent 工作质量的核心控制变量。笔记指出，长上下文会影响注意力分布和生成相关性，因此任务材料需要分阶段暴露，而不是无差别堆叠。

### 渐进披露用于人机协作

渐进披露不仅是 UI/UX 概念，也被迁移到 AI Agent 工作流中：先提供当前阶段最必要的信息，再随着任务推进补充细节。这有助于降低上下文噪声，让 Agent 更聚焦。

### 多 Agent 串行审阅

笔记提出用多个 Agent 进行连续审阅。其价值不在于简单重复，而在于让不同审阅轮次聚焦不同方向，例如架构边界、测试完备性、可读性或实现风险，从而形成更稳定的迭代改进链。
