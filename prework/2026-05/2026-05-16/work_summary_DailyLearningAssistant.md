## 主要工作主题

### 学习日报发布链路更完整

提交不仅新增 HTML 报告，还同步更新 manifest 和知识记录。这样日报不只是一个孤立页面，而是能进入首页导航、历史归档和长期知识索引的结构化内容。

### 邮件推送从设想变为可执行脚本

`send_daily_email.py` 将静态站点报告转换成邮件触达流程：从 `daily_report/manifest.json` 查找目标日期报告，用配置中的站点地址拼出报告链接，并发送 HTML 与纯文本双格式邮件。脚本还支持旧版单收件人配置兼容和 `--dry-run`，便于上线前验证。

### LLM 被用于个性化通知文案

邮件脚本可以调用配置中的 LLM API 生成每日问候和点评；失败时会回退到固定文案。这是一种实用的“AI 增强但不阻塞主流程”设计：个性化内容可以提升体验，但不应让报告发送依赖单点外部模型成功。

### 本地定时任务被纳入自动化运维

`setup_launchd.sh` 负责在 macOS 上安装 LaunchAgent，读取 `config.json` 中的 `schedule.daily_run_time`，并提供安装、卸载和状态查看能力。它把“每天手动运行脚本”转成“本机按时触发”的轻量调度方案。

## 可能涉及的知识点线索

- **静态站点 Manifest 契约**：页面文件、索引文件和首页导航之间需要稳定的发现契约。
- **邮件自动化与 SMTP 推送**：自动邮件需要配置隔离、收件人校验、HTML/Plain 双格式和失败提示。
- **本地任务调度**：launchd 通过 plist、StartCalendarInterval、日志路径和 WorkingDirectory 描述定时执行环境。
- **优雅降级**：LLM 文案生成失败时回退固定内容，避免非核心增强能力阻断日报发送。
- **配置样例与密钥隔离**：`config.example.json` 提供结构，不提交真实 `config.json`，这是自动化脚本可复用和安全落地的前提。
