## 详细关联描述

1. **依赖关系**
   - `多阶段自动化流水线` 依赖 `流水线前置条件`：日报链路由多个 Agent 任务串联，如果下游不验证上游产物，就可能在错序或空输入情况下继续运行。
   - `流水线前置条件` 依赖 `上游产物完整性校验`：前置条件需要落到具体文件检查上，例如确认 5 个 `work_summary_*.md` 均存在、非空且可读取。

2. **共同服务于某个目标**
   - `上游产物完整性校验`、`输入缺失时停止生成` 和 `防幻觉生成边界` 共同服务于可信内容生成：它们让系统在证据不足时暴露问题，而不是用看似完整的文本掩盖缺失。
   - `静态站点发布索引` 和 `知识日志归档` 共同服务于日报发布闭环：前者让页面可被首页导航发现，后者让月度知识记录保持可追踪。

3. **耦合关系**
   - `结构化输出契约` 与 `多阶段自动化流水线` 强耦合：每个阶段通过固定路径、固定文件名和固定内容结构向下一阶段交接。
   - `配置驱动自动化` 与 `结构化输出契约` 相关：配置化 LLM 调用需要稳定输入和输出边界，否则同一配置难以在多日自动化中复用。

4. **延伸关系**
   - `防护式自动化` 是从普通自动化生成任务延伸出来的可靠性设计：任务不再只关心生成结果，还要判断运行条件是否成立。
   - `few-shot 示例质量控制` 延伸到 `概念图谱`：高质量示例会改变下游 Agent 的输出分布，使结果更倾向于展示概念之间的关系。

5. **类比关系**
   - `概念图谱` 可类比为知识组织中的依赖图：它把孤立概念变成节点，把“依赖、耦合、共同目标、延伸”等说明变成边，帮助后续知识讲解按结构展开。

## 后续知识讲解建议

1. **自动化可靠性主线**
   - 讲解多阶段流水线为什么需要前置条件，以及文件存在、非空、可读取这三类检查分别防什么问题。
   - 结合本项目说明“停止生成”比“生成无输入占位结果”更适合正式日报链路的原因。

2. **LLM 工程主线**
   - 讲解防幻觉生成边界：如何通过证据来源、输入检查、禁止覆盖和禁止编造来约束 Agent 输出。
   - 讲解 few-shot 示例质量控制：示例的结构、粒度和关联描述如何影响后续模型输出。

3. **知识组织主线**
   - 讲解概念图谱与术语清单的区别：前者强调关系，后者只强调枚举。
   - 说明概念图谱如何服务下一阶段 `knowledge_explaination.md`：先确定节点和边，再展开生动讲解。

4. **发布工程主线**
   - 讲解静态站点发布索引中 `manifest.json` 的角色，以及它如何让新增日报进入首页导航。
   - 讲解知识日志归档如何与每日 HTML 报告形成互补：一个面向浏览，一个面向长期追踪。
