## 详细关联描述

1. **依赖关系**
   - `多阶段学习内容生成流水线` 依赖 `上游产物完整性校验`：下游概念提炼、知识讲解和 HTML 发布都需要确认前序材料真实存在，否则链路会把缺失输入放大成错误输出。
   - `静态站点发布闭环` 依赖 `发布契约`：新增 HTML 只是页面文件，只有同时更新 manifest 和知识日志，才算完成可发现、可归档的发布。

2. **共同服务于某个目标**
   - `上游产物完整性校验`、`输入缺失时停止生成` 和 `防幻觉生成边界` 共同服务于可信生成：它们把“证据不足”变成显式停止条件，而不是让 LLM 生成看似完整但不可追溯的内容。
   - `静态站点发布索引`、`机器可读导航元数据` 和 `知识日志归档` 共同服务于日报长期可用：前者让页面能被站点发现，后者让知识点能被月度复盘。

3. **耦合关系**
   - `发布契约` 与 `静态站点发布索引` 强耦合：manifest 中缺少新日报入口时，HTML 文件虽然存在，却可能无法成为首页默认显示或归档导航的一部分。
   - `自动化产物证据链` 与 `数据血缘` 耦合：证据链提供文件层面的顺序和来源，数据血缘解释最终日报内容如何从上游总结逐层加工而来。

4. **延伸关系**
   - `多阶段学习内容生成流水线` 延伸出 `自动化产物证据链`：一旦每个阶段都保留中间产物，流水线就不只是执行流程，也成为可审计的内容生产记录。
   - `概念图谱` 延伸到 `知识日志归档`：概念之间的关系可以帮助后续知识日志避免只记录孤立词条，而是保留学习路径。

5. **类比关系**
   - `manifest.json` 可类比为静态站点的目录索引：它不承载日报正文，却决定新增日报是否能被首页、归档或其他自动化入口发现。
   - `自动化产物证据链` 可类比为内容生产中的审计轨迹：它让读者和维护者能够从最终页面反查到概念、讲解和原始仓库总结。
