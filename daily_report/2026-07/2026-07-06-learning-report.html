Pseudorandom Number Generator Algorithm and Cycle Characteristics

精读文章 · 数学 / 计算机科学

伪随机数生成器的算法选择与周期特性

完整讲解

你电脑里的骰子其实是一本很厚很厚的“随机假面舞会名单”。它从一个种子出发，用数学公式算出下一个数字，再下一个，直到名单尽头才循环。不同算法就像不同版本的名册：有的只有一页，很快重复；有的厚到宇宙灭亡都用不完。游戏里，我们为什么关心它？因为如果你想重现一场精彩的AI决斗，必须用同一份名册。如果算法太短，仿真跑到半路序列就开始循环，相当于后半程在重演前半程，得出的胜率就是假的。好的算法（如梅森旋转）周期高达2的19937次方减1，即使在数千局仿真中也不会露出循环尾巴。选择和实现PRNG时，还要注意多线程里的“共享骰子”陷阱，否则一个线程乱翻名单，另一个线程的复现就崩了。

原理逻辑

  1. 伪随机数基于数学递推公式生成，给定种子即产生确定序列。
  2. 周期指序列不重复的长度，超过周期则循环。
  3. 多线程需独立随机数流，避免共享状态破坏复现。

具体例子

假设我们想重现一场海战仿真的1000局。如果用周期只有2^32≈43亿的线性同余发生器，每局消耗几万个随机数，在几万局内就可能回到起点，导致仿真后期不自觉地重复早期序列。换成周期为2^19937-1的梅森旋转，即使每秒用掉百亿个随机数，到宇宙热寂也不会循环。

常见误区

  • 只要固定种子，用任何一个随机函数都能完美复现。实际上不同算法序列不同，简单算法周期短、低位有规律。
  • 早期rand()等简单算法在大量采样后会露出非随机性，破坏仿真结果。

概念关联

关联概念关系说明

打破平局的传递性约束与无环偏好关系当用固定序列表作为次级标准时，这个序列可能依赖PRNG生成，理解周期能防止意外循环。

精读文章 · 数学 / 软件工程

仿真实验的统计功效与最小样本量确定

完整讲解

做一次仿真就像抛硬币检验公平性。你抛10次，得到7次正面，能不能说硬币不公平？不能，因为即便完全公平，10次中出现7次以上的概率也不太小。统计功效就是当硬币确实不公平时，你能拍胸脯发现它的可能性。要提高功效，就得增加抛硬币的次数。在游戏AI仿真中，我们要比较两套阵容的胜率，假设真实胜率差异只有5%（比如55%比50%），如果我们跑的局数太少，随机波动会淹没这个微小差异，结论就是“看运气”。功效分析会根据你愿意接受的错误概率（α=0.05）和希望达到的发现能力（1-β=0.8），计算出最少需要的仿真局数N。这样，你就不用盲目跑1000局，也不会因为只跑了100局就匆匆下结论说A不比B强。

原理逻辑

  1. 假设检验存在两类错误：α（假阳性）和β（假阴性），功效=1-β。
  2. 通过效应量指标（如Cohen's h），结合检验类型计算所需样本量。
  3. 自适应序贯实验可在仿真运行中动态判断是否达到显著，避免固定样本量的浪费。

具体例子

你想验证新配方的咖啡是否更提神，自己喝两天新咖啡，两天旧咖啡，结果新咖啡两天精神抖擞，旧咖啡一天打盹。这样的实验别人会笑你样本太少。统计功效就是那个懂行朋友，告诉你：“想证明新配方有效，至少需要连续测试30天，每天随机喝一种，再记录精力分数。”游戏AI仿真也一样，功效分析帮你敲定这个“至少多少次”的精确数字。

常见误区

  • 只要p值小于0.05，就证明结论可靠。实际上小样本下p<0.05也可能是假阳性。
  • 阴性结果不说明无差异，可能只是样本量不足导致功效太低。

概念关联

精读文章 · 数学 / 计算机科学 / 游戏AI

打破平局的传递性约束与无环偏好关系

完整讲解

当AI面对多个评分一样好的选项时，需要用次级规则来打破平局。这些规则就像一场多轮投票。如果投票机制设计得不好，可能出现“A比B好，B比C好，C却又比A好”的鬼打墙——这在数学上叫非传递性偏好，本质是一个有向循环。一旦决策逻辑里藏了这种循环，AI会在几个选项间反复横跳，或者做出看似不可理喻的行动。要避开这个陷阱，所有打破平局的次级规则必须构成一个严格全序关系，满足传递性：若A优于B，B优于C，则A必优于C。常见的保险做法是用一个固定的全局属性序列（如射程>血量>ID）建立一个树状比较，确保最终落在一个无环的链条上。传递性约束不仅是逻辑洁癖，更是保证AI行为一致、可解释的数学基石。

原理逻辑

  1. 传递性：若A>B且B>C则A>C，保证偏好无循环。
  2. 多标准平局打破可能隐含循环，需用严格全序结构。
  3. 若引入随机噪声打破平局，强度必须小到不破坏固有的传递性顺序。

具体例子

剪刀石头布就是一个经典的非传递性循环。如果AI在选择攻击目标时，距离相同，就用“血量少优先级高”打破平局，但血量相同又用“速度慢优先”，而速度相同又用“伤害高优先”……你可能在无意中组装出一个剪刀石头布式的循环：目标A血量少，B速度慢，C伤害高，且A的速度比B快，B的伤害比C低，而C的血量比A少，最终绕成一团。这时AI会像钟摆一样来回摇摆。

常见误区

  • 只要给每个选项打分，分数最高的胜出，就一定不会循环。实际上，若打分采用动态变量或反馈，仍可能引入循环。
  • 用排名、对手上一次行动等非传递性标准打破平局，会悄然钻进循环。

概念关联

关联概念关系说明
定量决策模型中的平局打破策略为各种性格底牌套上逻辑紧箍咒，确保无论何种规则都经传递性检查。
伪随机数生成器的算法选择与周期特性当用实体ID或固定序列作为次级标准时，序列生成依赖PRNG，理解周期可防止隐藏循环。
